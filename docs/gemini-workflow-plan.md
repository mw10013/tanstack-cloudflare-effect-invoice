# Plan: Gemini Invoice Extraction Workflow

## Goal

New workflow `GeminiInvoiceExtractionWorkflow` that sends the file (PDF or image) directly to Gemini 2.5 Flash for structured invoice extraction. Replaces the current two-step pipeline (toMarkdown → gpt-oss-120b) with a single Gemini call.

## Current State

- `InvoiceExtractionWorkflow` (keep for reference, don't delete):
  1. `load-pdf` — fetch from R2
  2. `convert-pdf-to-markdown` — `env.AI.toMarkdown()`
  3. `save-markdown` — write markdown to Invoice table
  4. `extract-invoice-json` — `runInvoiceExtractionViaGateway()` (gpt-oss-120b)
  5. `save-invoice-json` — write invoiceJson to Invoice table
- `onInvoiceUpload` skips non-PDFs at line 151: `if (upload.contentType !== "application/pdf") return`
- Accepted types: `application/pdf`, `image/png`, `image/jpeg`, `image/webp`, `image/gif`
- `GOOGLE_AI_STUDIO_API_KEY` env var ready

## New Workflow: `GeminiInvoiceExtractionWorkflow`

### Steps

1. **`load-file`** — fetch file bytes from R2 (same as current `load-pdf` but renamed since it handles images too)
2. **`extract-invoice`** — single Gemini call: send file bytes + extraction prompt → get structured JSON back
3. **`save-invoice-json`** — write invoiceJson to Invoice table

No markdown step. Gemini handles the visual parsing internally.

### Gemini Request

```typescript
const url = `https://gateway.ai.cloudflare.com/v1/${accountId}/${gatewayId}/google-ai-studio/v1/models/gemini-2.5-flash:generateContent`;

const response = await fetch(url, {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "x-goog-api-key": googleAiStudioApiKey,
    "cf-aig-authorization": `Bearer ${aiGatewayToken}`,
  },
  body: JSON.stringify({
    contents: [{
      parts: [
        { text: EXTRACTION_PROMPT },
        { inlineData: { mimeType: contentType, data: base64FileData } },
      ],
    }],
    generationConfig: {
      responseMimeType: "application/json",
      responseSchema: InvoiceExtractionJsonSchema,
    },
  }),
});
```
